## Rationale

## Security Considerations

Every RFC MUST include this section, even if the answer is "This
RFC introduces no new security considerations."  Address at minimum:

- Authentication and authorization implications
- Data confidentiality and integrity
- Denial-of-service vectors
- PII exposure risks
